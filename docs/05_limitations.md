# Limitations

## Synthetic Data

Synthea data are synthetic and do not represent real patients. Although the data are useful for demonstrating EHR analytics workflows, results should not be interpreted as clinically valid or generalizable.

## Coding and Encounter Definitions

Readmission classification depends on how encounters, dates, and encounter classes are represented in the source data. The project will document assumptions and validation checks as the cohort logic is developed.

## Missing Clinical Detail

Synthetic EHR data may not include all clinical, social, operational, or discharge planning factors relevant to readmission risk in real-world settings.

## Model Interpretation

Any statistical or machine learning model developed in this project will be used to demonstrate analytic methods. Model performance should not be treated as evidence of deployable clinical utility.

## Portfolio Scope

This project is designed as a one-month portfolio project. It prioritizes clear cohort logic, reproducible analysis, and professional communication over production-grade data infrastructure.
