# Mock IRB Summary

## Project Type

This is a portfolio demonstration using synthetic EHR data generated by Synthea. It is not human subjects research and does not involve real patient records.

## Study Objective

The project demonstrates a retrospective analytic workflow for identifying all-cause inpatient readmission within 30 days after a first eligible acute inpatient hospitalization and describing associated patient and encounter characteristics.

## Data Privacy

No protected health information is included in this repository. Synthea data are synthetic and are not derived from actual individual patients. Large data files should not be committed to version control.

## Risk Assessment

The privacy risk is minimal because the project does not use real patient data. The main project risks are analytic misclassification, unclear cohort definitions, and overinterpretation of synthetic results.

## Ethical and Reporting Considerations

Results from synthetic data should not be presented as clinical evidence. Any final report should clearly state that findings are for workflow demonstration and portfolio purposes only.

## Data Handling Plan

Synthetic source data, local databases, and generated outputs should be stored locally and excluded from version control unless they are small, non-sensitive, and intentionally included as documentation examples.
