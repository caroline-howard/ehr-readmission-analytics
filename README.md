# EHR Readmission Analytics

This repository is a one-month healthcare data analytics portfolio project using Synthea synthetic electronic health record data. The project demonstrates an end-to-end retrospective research analytics workflow for 30-day hospital readmission, from cohort definition and SQL-based data preparation through validation, statistical analysis, reporting, and a lightweight project app.

The project is designed for healthcare research analytics roles that involve EHR data, cohort construction, SQL, analytic planning, reproducibility, data quality checks, statistical interpretation, and manuscript-style outputs.

## Project Goal

Develop a reproducible retrospective analytics workflow to study 30-day hospital readmission after an index inpatient encounter using synthetic EHR data generated by Synthea.

This repository does not contain real patient data. Synthea data are synthetic and should be treated as mock EHR data for portfolio demonstration only.

## Planned Workflow

1. Define the research question, target population, index encounter, outcome, and key covariates.
2. Load and inspect Synthea EHR tables.
3. Build SQL queries for cohort selection, encounter linkage, diagnosis summaries, and readmission outcome construction.
4. Validate cohort counts, missingness, date logic, and readmission classification.
5. Conduct descriptive and statistical analyses in notebooks.
6. Produce manuscript-style tables, figures, and a final analytic report.
7. Build a small Gradio app to present project context and selected outputs.

## Repository Structure

```text
docs/       Project overview, analytic plan, data dictionary, IRB-style summary, and limitations
sql/        SQL scripts for cohort construction, feature engineering, and validation
notebooks/  Analysis notebooks for data exploration, modeling, and outputs
outputs/    Generated tables, figures, and validation artifacts
report/     Manuscript-style report materials
app/        Lightweight Gradio app for portfolio presentation
```

## Current Status

Week 1 setup is focused on repository structure, project documentation, and analytic planning. No synthetic data files, analysis outputs, or fake results have been added.

## Data Source

This project is intended to use synthetic EHR data from Synthea. Synthea generates realistic but non-real patient records for testing, training, and demonstration. Any downloaded or generated data should be stored outside version control or in ignored local directories.

## Reproducibility

The project will use Python, SQL, and notebook-based analysis. Dependencies are listed in `requirements.txt`. Future work will add clear instructions for obtaining Synthea data, loading tables, running SQL scripts, and reproducing outputs.
